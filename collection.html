Lesson 1: Create Database

Create a new clinic database:

CREATE DATABASE clinic_db;

Lesson 2: Use Database

Select the database before creating tables:

USE clinic_db;

Lesson 3: Drop Database

Delete database (be careful!):

DROP DATABASE clinic_db;

Lesson 4: Read Only Mode

Make database read-only:

ALTER DATABASE clinic_db READ ONLY = 1;

Turn off read-only:

ALTER DATABASE clinic_db READ ONLY = 0;

Lesson 5: Create Patients Table

CREATE TABLE patients(
    patient_id INT,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    date_of_birth DATE
);

Lesson 6: Show Data

SELECT * FROM patients;

Lesson 7: Rename Table

RENAME TABLE patients TO patient_record;

Lesson 8: Alter Table

Add column:

ALTER TABLE patient_record
ADD phone VARCHAR(15);

Rename column:

ALTER TABLE patient_record
RENAME COLUMN phone TO phone_number;

Modify column size:

ALTER TABLE patient_record
MODIFY COLUMN phone_number VARCHAR(20);

Move column position:

ALTER TABLE patient_record
MODIFY COLUMN phone_number VARCHAR(20)
AFTER first_name;

Lesson 9: Drop Table

DROP TABLE patient_record;

Lesson 10: Update Data

Update a patient phone number by id:

    UPDATE patient_record
    SET phone_number = '09-555-1234'
    WHERE patient_id = 1;

Update multiple fields:

    UPDATE patient_record
    SET first_name = 'Aye', last_name = 'Tun'
    WHERE patient_id = 2;

Lesson 11: Delete Data

Delete one patient record safely with a condition:

    DELETE FROM patient_record
    WHERE patient_id = 3;

Delete all rows in a table:

    DELETE FROM patient_record;

Lesson 12: Date & Time Functions

Create a table for date and time testing:

USE projectdb;

CREATE TABLE timedb(
  my_date DATE,
  my_time TIME,
  my_datetime DATETIME
);

Insert the current date, time, and date-time:

INSERT INTO timedb(my_time, my_date, my_datetime)
VALUES (CURRENT_TIME(), CURRENT_DATE(), NOW());

View the results:

SELECT * FROM timedb;

CURRENT_TIME() returns only the current time.

CURRENT_DATE() returns only the current date.

NOW() returns the current date and time together.

Lesson 13: Unique Constraint

Make sure each email can appear only once:

CREATE TABLE staff(
  staff_id INT,
  email VARCHAR(100) UNIQUE,
  full_name VARCHAR(100)
);

Insert sample staff records:

INSERT INTO staff(staff_id, email, full_name)
VALUES
  (1, 'nandar@clinic.com', 'Nandar Kyaw'),
  (2, 'thandar@clinic.com', 'Thandar Hlaing'),
  (3, 'aungmin@clinic.com', 'Aung Min');

Try a duplicate email (this will fail because of UNIQUE):

    INSERT INTO staff(staff_id, email, full_name)
    VALUES (4, 'nandar@clinic.com', 'Duplicate Email');

Add a UNIQUE rule after the table exists:

ALTER TABLE staff
ADD CONSTRAINT unique_staff_email UNIQUE (email);

unique_staff_email is the constraint name, so you can manage it later.

In MySQL, a UNIQUE constraint is stored as a unique index.

Remove the UNIQUE rule (using the constraint name):

    ALTER TABLE staff
    DROP INDEX unique_staff_email;

List indexes (to see UNIQUE constraints):

SHOW INDEX FROM staff;

Rename the UNIQUE index (constraint name):

    ALTER TABLE staff
    RENAME INDEX unique_staff_email TO unique_staff_email_v2;

Lesson 14: NOT NULL Constraint

Require a value so the column cannot be empty:

    CREATE TABLE appointments(
      appointment_id INT,
      patient_name VARCHAR(100) NOT NULL,
      appointment_date DATE NOT NULL,
      notes VARCHAR(200)
    );

Add NOT NULL to an existing column:

    ALTER TABLE appointments
    MODIFY COLUMN patient_name VARCHAR(100) NOT NULL;

Remove NOT NULL if you want to allow empty values:

    ALTER TABLE appointments
    MODIFY COLUMN patient_name VARCHAR(100) NULL;

Try inserting without a required value (this will fail):

    INSERT INTO appointments(appointment_id, patient_name, appointment_date)
    VALUES (1, NULL, '2026-02-10');

Lesson 15: CHECK Constraint

Validate values before saving a row:

    CREATE TABLE payments (
      payment_id INT,
      amount DECIMAL(10,2) CHECK (amount > 0)
    );

Add a CHECK rule after the table exists:

    ALTER TABLE payments
    ADD CONSTRAINT chk_amount_positive CHECK (amount > 0);

Try a value that breaks the rule (this will fail):

        INSERT INTO payments(payment_id, amount)
        VALUES (1, -50.00);

chk_amount_positive is just a name; you can choose any valid name.

MySQL enforces CHECK constraints starting from version 8.0.16.

Lesson 16: Primary Key

A primary key uniquely identifies each row and prevents duplicate ids.

    CREATE TABLE patients_pk(
      patient_id INT PRIMARY KEY,
      full_name VARCHAR(100) NOT NULL,
      date_of_birth DATE
    );

Add a primary key after the table exists:

    ALTER TABLE patients_pk
    ADD CONSTRAINT pk_patients PRIMARY KEY (patient_id);

If you want to remove the primary key, use the constraint name:

            ALTER TABLE patients_pk
            DROP CONSTRAINT pk_patients; -- Uses the name you gave it

Insert a valid row:

            INSERT INTO patients_pk(patient_id, full_name, date_of_birth)
            VALUES (1, 'Hnin Thandar', '1999-07-18');

Try a duplicate id (this will fail):

          INSERT INTO patients_pk(patient_id, full_name, date_of_birth)
          VALUES (1, 'Duplicate Patient', '2000-01-01');

Lesson 17: AUTO_INCREMENT

Let MySQL generate a new id for each row automatically.

    CREATE TABLE patients_auto(
      patient_id INT PRIMARY KEY AUTO_INCREMENT,
      full_name VARCHAR(100) NOT NULL,
      date_of_birth DATE
    );

    INSERT INTO patients_auto(full_name, date_of_birth)
    VALUES ('May Thu', '1998-09-12');

    INSERT INTO patients_auto(full_name, date_of_birth)
    VALUES ('Aung Aung', '2001-04-30');

You can set a starting value with AUTO_INCREMENT = 100 on the table.

Auto ids can skip numbers if rows are deleted.

Set the starting value after the table exists:

    ALTER TABLE patients_auto AUTO_INCREMENT = 100;

Insert another row and view the results:

            INSERT INTO patients_auto (full_name, date_of_birth)
            VALUES ('New Patient', '2000-01-01');

            SELECT * FROM patients_auto;
